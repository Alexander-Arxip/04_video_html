НАЧАЛО

1. Создание папки Git
Теперь давайте создадим новую папку для нашего проекта:
mkdir myproject
cd myproject

2. Инициализировать Git
Перейдя в нужную папку, вы можете инициализировать Git в этой папке:
git init
Initialized empty Git repository in /Users/user/myproject/.git/ Вы только что создали свой первый репозиторий Git!
Теперь Git знает, что ему следует следить за папкой, в которой вы его инициировали.
Git создает скрытую папку для отслеживания изменений.


3. СОЗДАДИМ ФАЙЛ index.html
И сохраните его в нашей новой папке как index.html.
Давайте вернемся к терминалу и перечислим файлы в нашем текущем рабочем каталоге:
ls index.html
Затем мы проверяем Git statusи смотрим, является ли он частью нашего репозитория:
git status
On branch master
Теперь Git знает об этом файле, но не добавил его в наш репозиторий!
Файлы в папке вашего репозитория Git могут находиться в одном из двух состояний:
Отслеживается — файлы, о которых Git знает и которые добавляются в репозиторий.
Неотслеживаемые — файлы, находящиеся в вашем рабочем каталоге, но не добавленные
репозиторий.
Когда вы впервые добавляете файлы в пустой репозиторий, все они не отслеживаются. Чтобы Git
мог их отслеживать, вам необходимо подготовить их или добавить в промежуточную среду.


4. Одной из основных функций Git являются концепции промежуточной среды и фиксации.
Во время работы вы можете добавлять, редактировать и удалять файлы. Но всякий раз, когда вы достигаете важной вехи или завершаете часть работы, вам следует добавлять файлы в промежуточную среду.
Промежуточные файлы — это файлы, которые готовы к фиксации в репозитории, над которым вы работаете. Вы узнаете больше об этом commitв ближайшее время.
На данный момент мы закончили работу с index.html. Итак, мы можем добавить его в промежуточную среду:
git add index.html
Файл должен быть проиндексирован . Давайте проверим статус::
git status On branch master

5. Git добавить более одного файла
Вы также можете разместить более одного файла одновременно. Добавим в нашу рабочую папку еще 2 файла. Снова воспользуйтесь текстовым редактором.
Файл README.md, описывающий репозиторий (рекомендуется для всех репозиториев):
ДОБАВИМ Базовая внешняя таблица стилей ( bluestyle.css):
Теперь добавьте все файлы из текущего каталога в промежуточную среду:
git add --all
