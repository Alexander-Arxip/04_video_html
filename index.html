Работы с гит. Добавление файлов и репозиториев

1. Создание папки Git
Теперь давайте создадим новую папку для нашего проекта:
mkdir myproject
cd myproject

2. Инициализировать Git
Перейдя в нужную папку, вы можете инициализировать Git в этой папке:
git init
Initialized empty Git repository in /Users/user/myproject/.git/ Вы только что создали свой первый репозиторий Git!
Теперь Git знает, что ему следует следить за папкой, в которой вы его инициировали.
Git создает скрытую папку для отслеживания изменений.


3. СОЗДАДИМ ФАЙЛ index.html
И сохраните его в нашей новой папке как index.html.
Давайте вернемся к терминалу и перечислим файлы в нашем текущем рабочем каталоге:
ls index.html
Затем мы проверяем Git statusи смотрим, является ли он частью нашего репозитория:
git status
On branch master
Теперь Git знает об этом файле, но не добавил его в наш репозиторий!
Файлы в папке вашего репозитория Git могут находиться в одном из двух состояний:
Отслеживается — файлы, о которых Git знает и которые добавляются в репозиторий.
Неотслеживаемые — файлы, находящиеся в вашем рабочем каталоге, но не добавленные
репозиторий.
Когда вы впервые добавляете файлы в пустой репозиторий, все они не отслеживаются. Чтобы Git
мог их отслеживать, вам необходимо подготовить их или добавить в промежуточную среду.


4. Одной из основных функций Git являются концепции промежуточной среды и фиксации.
Во время работы вы можете добавлять, редактировать и удалять файлы. Но всякий раз, когда вы достигаете важной вехи или завершаете часть работы, вам следует добавлять файлы в промежуточную среду.
Промежуточные файлы — это файлы, которые готовы к фиксации в репозитории, над которым вы работаете. Вы узнаете больше об этом commitв ближайшее время.
На данный момент мы закончили работу с index.html. Итак, мы можем добавить его в промежуточную среду:
git add index.html
Файл должен быть проиндексирован . Давайте проверим статус::
git status On branch master

5. Git добавить более одного файла
Вы также можете разместить более одного файла одновременно. Добавим в нашу рабочую папку еще 2 файла. Снова воспользуйтесь текстовым редактором.
Файл README.md, описывающий репозиторий (рекомендуется для всех репозиториев):
ДОБАВИМ Базовая внешняя таблица стилей ( bluestyle.css):
Теперь ДОБАВИМ все файлы из текущего каталога в промежуточную среду:
git add --all
git status
Теперь все три файла добавлены в промежуточную среду, и мы готовы создать наш первый файл commit.

6. Git-коммит Поскольку мы завершили нашу работу, мы готовы перейти stageк commit нашему репо.
Добавление коммитов позволяет отслеживать наш прогресс и изменения во время работы. Git учитывает каждую commitточку изменения или «точку сохранения».
Это точка в проекте, к которой вы можете вернуться, если обнаружите ошибку или захотите внести изменения.
Когда мы commit, мы всегда должны включать сообщение .
Добавляя четкие сообщения к каждому элементу commit, вы (и другие) сможете легко увидеть, что и когда изменилось.
git commit -m "это мой ПЕРВЫЙ коммит к моим файлам"
Краткие флаги состояния:
?? - Неотслеживаемые файлы
A — Файлы, добавленные в сцену.
М - Измененные файлы
D - Удаленные файлы
Журнал коммитов Git
Чтобы просмотреть историю коммитов репозитория, можно использовать log команду:
git log
когда вам понадобится помощь в запоминании конкретной опции команды, вы можете использовать :
git --help

7. Работа с ветками Git
В Git a branch— это новая/отдельная версия основного репозитория.
Допустим, у вас большой проект, и вам необходимо обновить дизайн на нем.
Ветви позволяют работать над разными частями проекта, не затрагивая основную ветку.
Когда работа будет завершена, ветку можно объединить с основным проектом.
Вы даже можете переключаться между ветками и работать над разными проектами, не мешая друг другу.
Ветвление в Git очень легкое и быстрое!
Новая ветка Git
Давайте добавим некоторые новые функции на нашу index.html страницу.
Мы работаем в нашем локальном репозитории и не хотим нарушать или, возможно, разрушать основной проект.
Итак, мы создаем новый branch:
git branch hello-world-images
Теперь мы создали новый branch под названием " hello-world-images"
Давайте подтвердим, что мы создали новый branch:
git branch
hello-world-images
* master
Мы видим новую ветку с именем «hello-world-images», но рядом с ней *указано master, что в данный момент мы находимся в этом файле branch.
checkout— это команда, используемая для извлечения файла branch. Перемещаем нас из текущего branch, в тот, который указан в конце команды:
git checkout hello-world-images
Switched to branch 'hello-world-images'
Теперь мы переместили нашу текущую рабочую область из основной ветки в новую. branch
Откройте свой любимый редактор и внесите некоторые изменения.
Для этого примера мы добавили ФОТО (img_hello_world.jpg) в рабочую папку и строку кода в index.html файл:
git add --all


8. Теперь давайте посмотрим, что произойдет, если мы изменим ветку на master
git checkout master
Мы добавили изображение в эту ветку, поэтому давайте перечислим файлы в текущем каталоге:
ls
Теперь давайте посмотрим, что произойдет, если мы изменим ветку на master
git checkout master
подготовьте файл и зафиксируйте:
git add index.html
Теперь ЕСЛИ ЗАХОЧЕМ объединяем текущую ветку (master) с любой другой ВЕТКОЙ:
ФУНКЦИЯ
git merge emergency-fix
ПОТОМ НАДО зафиксировать эту ветку: git add --all git commit -m "я приложил новое ФОТО"
Теперь мы можем ОБНОВИТЬ index.html и проверить статус:
git add index.html
git status
мы можем использовать команду commit для завершения слияния:
git commit -m "завершаем слияние"
